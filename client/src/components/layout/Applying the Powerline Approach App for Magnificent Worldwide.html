1. Revised Placement Algorithm

Per your requirements, the placement algorithm has been corrected to ensure that new members are always placed on the outer edges of the binary tree, while still maintaining the binary structure required for Talk Fusion's compensation plan.

Updated Binary Placement Algorithm

/** * Places new members on the outer edges of the binary tree structure * - New promoters are only placed on the outside of either right or left legs * - Members are placed on the outermost position of their sponsor's preferred leg * - Everyone builds two legs, but powerline visualization focuses on outer positions */ function placeInBinary(newMember, sponsor) { // Step 1: Determine which leg the sponsor prefers for this placement const preferredLeg = sponsor.placementPreference || getDefaultLeg(sponsor); // Step 2: Find the outermost position in that leg const placementPosition = findOutermostPositionInLeg(sponsor.memberId, preferredLeg); // Step 3: Place the new member at that position createBinaryRelationship(placementPosition.parentId, newMember.memberId, placementPosition.position); // Step 4: Log the placement for tracking logPlacement(newMember.memberId, placementPosition.parentId, placementPosition.position); return { success: true, position: placementPosition, leg: preferredLeg }; } /** * Finds the outermost position in the specified leg * This ensures we only place on the outside edges of the tree */ function findOutermostPositionInLeg(sponsorId, preferredLeg) { // Start with the sponsor's position in their leg let currentMemberId = sponsorId; // Find the "main line" down the preferred leg while (true) { // Check if there's already someone in the preferred position const childId = getBinaryChildId(currentMemberId, preferredLeg); if (childId) { // Move down to that position and continue currentMemberId = childId; } else { // Found an empty spot on the outer edge of the preferred leg return { parentId: currentMemberId, position: preferredLeg }; } } } /** * Determines the default leg based on sponsor's preference or team balance */ function getDefaultLeg(sponsor) { // If sponsor has a saved preference, use that if (sponsor.defaultLeg) { return sponsor.defaultLeg; } // Otherwise, check team balance const leftCount = countLegMembers(sponsor.memberId, 'left'); const rightCount = countLegMembers(sponsor.memberId, 'right'); // Place on the leg with fewer members for better balance return leftCount <= rightCount ? 'left' : 'right'; }

Visual Representation of Correct Placement Logic

Sponsor (S) / \ / \ Member1 (M1) Member2 (M2) / / \ / / \ Member3 (M3) Member4 (M4) Member5 (M5) | | | | Member6 (M6) Member7 (M7)

In this corrected example:

  1. When S sponsors new members:
    • First member goes to position M1 (left leg)
    • Second member goes to position M2 (right leg)
  2. When M1 sponsors new members:
    • Their new member goes to M3 (left outer position)
    • Next member would go under M3 as M6 (continuing down the left outer edge)
  3. When M2 sponsors new members:
    • First member could go to M4 (left under M2)
    • Second member could go to M5 (right under M2)
    • If M2 prefers the right leg, new members would be placed under M5 as M7 (continuing down the right outer edge)

The key principle is that members are placed on the outermost available position of their sponsor's preferred leg, creating a straight line down each side of the binary structure.
